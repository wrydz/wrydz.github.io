1 前言

很早就想写博客了，不过一直没有真正落实下来。最近花了些时间，把博客建立起来了。本博文就这个过程作简要记录，以备查询。

2 hexo安装

hexo是基于node的静态博客系统，所以必须先配置好node环境。

2.1 node安装

node有3种安装方式: 1.直接命令行安装，据说此种方式安装的node有些瑕疵。 

sudo apt-get install node

2.在官网下载编译好的二进制文件 我使用的就是这种方式，登陆官网 https://nodejs.org/en/ 下载最新版本。 

# cd到node文件目录
xz -d node-v6.6.0-linux-x64.tar.xz
tar -xvf node-v6.6.0-linux-x64.tar
# 把解压的node文件夹移动到你喜欢的位置
mv node-v6.6.0-linux-x64 /path-to-you/node-v6.6.0-linux-x64
# 建立软连接
ln -s /path-to-you/node-v6.6.0-linux-x64/node /usr/local/bin/node
ln -s /path-to-you/node-v6.6.0-linux-x64/npm  /usr/local/bin/npm
# 至此node安装已经完成，使用命令验证是否安装成功,若显示当前版本号则成功
node -v
npm -v

3.下载源码编译安装 由于我没有使用这种方式安装过，就不多说了。

2.2 hexo安装

以下是hexo的安装及常用命令： 

# 安装hexo客户端
npm install hexo-cli -g
# 进入一个写博客的空目录
cd xxxx
# 初始化hexo
hexo init
# 生成静态文件
hexo g
# 部署文件
hexo d
# 启动本地服务，进行文章预览
hexo s

3 spacemacs配置

在debian中，我使用spacemacs来写博客，需要一些配置。

3.1 blog-mode

blog-admin是emacs中的一个用来管理博客的包，支持hexo，org-page，nikola，由CodeFalling编写维护，详细内容请看github https://github.com/CodeFalling/blog-admin

emacs中的配置 

(use-package blog-admin
  :init
  (progn
    (setq blog-admin-backend-path "~/static-blog")
    (setq blog-admin-backend-type 'hexo)
    (setq blog-admin-backend-new-post-in-drafts t) ;; create new post in drafts by default
    (setq blog-admin-backend-new-post-with-same-name-dir t) ;; create same-name directory with new post
    (add-hook 'blog-admin-backend-after-new-post-hook 'find-file)
    ))

3.2 org-markdown

hexo不直接支持org，需要安装插件把org转换成html，实际转换工具还是emacs。 

npm install https://github.com/CodeFalling/hexo-renderer-org#emacs --save

修改hexo配置文件_config.yml 

# emacs
org:
  emacs: '/usr/bin/emacs'
  common: |
          #+OPTIONS: num:nil
          #+OPTIONS: toc:nil
          #+BIND: org-html-postamble \"Last Updated %C.</br>Render by <a href='https://github.com/CodeFalling/hexo-renderer-org'>hexo-renderer-org</a> with %c\"
  cachedir: './hexo-org-cache/'
